1. SERVICE OVERVIEW
================================================================================

The License Service provides a REST API for managing software licenses, accounts,
entitlements, and customer data. It supports multiple API versions (v1, v3, v4)
with different capabilities per version.

Authentication: OAuth2 Bearer Token (required for all endpoints)
Base URL: Configured via BASE_URL environment variable
API Versions: 1.0, 3.0, 4.0 (passed as query param: ?api-version=X.0)

The service is composed of 3 sub-services:
  - License Controller: CRUD operations on licenses
  - Account Controller: Account-level queries, features, entitlements, cache
  - Customer Controller: Customer/user management (v1 only)

3. ENDPOINT DETAILS
================================================================================

--------------------------------------------------------------------------------
LICENSE ENDPOINTS (v3/v4)
--------------------------------------------------------------------------------

ENDPOINT 1: GET /licenses/v{ver}/License/account/{accountId}
  - Response 200: Array of license objects
    [{ accountId, licenseId, deviceId, coreFeature, ... }]
  - Non-existent account: Returns 200 with empty array []

ENDPOINT 2: GET /licenses/License/{licenseId}/feature/{featureName}?api-version=X
  - Query Params: getUsagePlan (boolean, default false)
  - Response 200: { accountId, featureId, startDate, endDate, limit, limitUnit }
  - Error: 404 (non-existent license/feature)

ENDPOINT 3: GET /licenses/License/{licenseId}?api-version=X
  - Response 200: { accountId, id, accountName, features[], sku, productName,
                    emsVersion, ... }
  - Error: 404 (non-existent license)

ENDPOINT 4: POST /licenses/License?api-version=X
  - Request Body:
    { accountId, coreFeature, deviceId, sku, licenseType, productName,
      productVersion }
  - Response 200: Created license object
  - Error: 400/500 (missing required fields, invalid accountId)

ENDPOINT 5: DELETE /licenses/License/{licenseId}?api-version=X
  - Response: 200/204 (success, idempotent — returns 204 for non-existent)
  - Error: 401 (unauthorized)

ENDPOINT 6: PUT /licenses/License/updateToken?api-version=X
  - Request Body: { licenseId, oldAccessToken }
  - Response 200: License object with refreshed token
  - Note: oldAccessToken is optional (ignored by server)
  - Error: 400/500 (empty licenseId)

ENDPOINT 7: PUT /licenses/License/renew?api-version=X
  - Request Body: { licenseId }
  - Response 200: Renewed license object
  - Error: 400/404/500 (non-existent or empty licenseId)

ACCOUNT ENDPOINTS (v3/v4)
--------------------------------------------------------------------------------

ENDPOINT 10: GET /licenses/Account?feature=X&licenseType=Y&api-version=X
  - Response 200: Array of account objects
    [{ id, name, licenseType, sku, productName, features[] }]
  - Error: 500 (invalid licenseType like 999)

ENDPOINT 11: GET /licenses/Account/{accountId}/feature/{featureName}?licenseType=Y
  - Response 200: { accountId, featureId, startDate, endDate, limit, limitUnit }
  - Error: 500 (non-existent accountId)

ENDPOINT 12: POST /licenses/Account/cache/reset?api-version=X
  - Request Body: { userIds: ["uuid1", "uuid2"] }
  - Response: 200/204 (success)

ENDPOINT 13: GET /licenses/Account/entitlements/{accountId}?api-version=4.0
  - Response 200: Boolean value ("true" or "false")

ENDPOINT 14: GET /licenses/Account/entitlements/{entitlementId}/users/{userId}
  - Response 200: { accountId, organisationName, userEmail, firstName,
                    lastName, country }
  - Error: 404 (invalid format for entitlementId or userId)

  - Response 200: Array of customers [{ id, name, users[] }]

ENDPOINT 18: GET /licenses/Customer/{customerId}/products?api-version=1.0

4. HOW IT WORKS (Architecture)
================================================================================

  STEP-BY-STEP: What happens when tests run
  ──────────────────────────────────────────

  Step 1: SETUP
    - global-setup.ts checks that environment variables are set (.env file)
    - ApiBaseTest.ts gets an OAuth2 token via TokenManager.ts
    - Creates API clients: LicenseClient, LicenseAccountClient,
      LicenseCustomerClient (for v1, v3, v4)

  Step 2: LICENSE TESTS (v3/v4)
    - GET    /License/account/{accountId}  → List all licenses for an account
    - GET    /License/{licenseId}/feature/{featureName} → Get feature details
    - GET    /License/{licenseId}          → Get license details
    - POST   /License                      → Create a new license
    - DELETE /License/{licenseId}          → Delete a license
    - PUT    /License/updateToken          → Refresh the license token
    - PUT    /License/renew               → Renew an existing license

  Step 3: LICENSE TESTS (v1)
    - GET    /License/{customerId}/product/{sku} → Request license for user
    - GET    /License/{licenseId}               → Check license validity
    - POST   /License/{licenseId}               → Refresh a license
    - DELETE /License/{licenseId}               → Release a license

  Step 4: ACCOUNT TESTS (v3/v4)
    - GET  /Account?feature=X&licenseType=Y    → Get accounts by feature/type
    - GET  /Account/{accountId}/feature/{name} → Get feature for an account
    - POST /Account/cache/reset                → Reset cache for user IDs
    - GET  /Account/entitlements/{accountId}   → Check entitlements (v4 only)
    - GET  /Account/entitlements/{id}/users/{userId} → User entitlement (v4)

  Step 5: CUSTOMER TESTS (v1)
    - GET /v1/user/{email}                → Get user by email
    - GET /Customer/listall               → List all customers
    - GET /Customer/list                  → List customers (filtered)
    - GET /Customer/{customerId}/products → Get products for a customer

  CLEANUP: Create tests always delete the licenses they created afterward.

  api/clients/license/

8. TROUBLESHOOTING
================================================================================

  Problem                            | Solution
  ───────────────────────────────────+──────────────────────────────────────
  401 on all license calls           | Check .env token credentials
  500 on GET Account                 | Likely invalid licenseType (use 1)
  500 on v1 Request License          | Customer/SKU doesn't exist
  404 on GET Feature                 | License ID or feature name invalid
  204 on DELETE (unexpected)         | License already deleted (idempotent)
  Create tests leave orphan licenses | Check cleanup (delete) at end of test
  Empty array on GET account license | AccountId not found (returns 200 + [])
